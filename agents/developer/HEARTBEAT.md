On each heartbeat:

1. **Pull your queue** — Issues assigned to you, sorted by:
   (a) Board-assigned priority=high
   (b) CTO-assigned with explicit `due` label
   (c) Bug label (production breakage)
   (d) Everything else FIFO

2. **Pick the top Issue.** Read its parent + grandparent Issues for context
   before touching any code.

3. **If it's a code change Issue:**
   - Read `shared/brand/vantyx.md` (and the per-client file if applicable).
   - Read the existing component / skill code under
     `github.com/Rfog88/vantyx-web-os` (clone read-only).
   - Implement the smallest change that solves the Issue. Do not refactor
     surrounding code unless explicitly asked.
   - Run the build locally: `pnpm build` (or `npm run build`).
   - Run Lighthouse mobile on the affected page; must be ≥85.
   - Commit with message format: `WEB-<issue>: <one-line summary>`.
   - Open a PR to vantyx-web-os; comment on the Paperclip Issue:
     "PR: <url> — ready for CTO review."

4. **If it's a deploy Issue (Phase 1.5):**
   - Pull the demo build directory generated by `template-fill`.
   - Verify `site.config.ts` is filled (no `TODO` markers, no empty fields).
   - Run `vercel --prod --token $VERCEL_TOKEN --scope rfog88s-projects --name preview-<lead-slug>`.
   - Alias to `<lead-slug>.preview.<your-domain>`.
   - Comment the live URL on the lead's Issue.

5. **If it's a bug Issue:**
   - Reproduce locally first. If you can't reproduce, comment "cannot
     reproduce — need: <X>" and assign back to CTO.
   - Write a regression test before fixing.
   - Fix; verify the test passes; PR as above.

6. **If your queue is empty:**
   - Read `shared/lessons.md` for any new entries that suggest a code-level
     fix (e.g., a recurring failure mode). Propose an Issue to CTO.
   - Otherwise, stay idle. Do not invent work.

If a heartbeat run itself fails (build broken, Vercel token rejected, etc.):
`diagnose-why-work-stopped` first; comment the diagnosis on the Issue; if you
can't resolve in this heartbeat, comment "@cto blocked: <reason>" and move
on. Do not retry the same failure in a loop.
